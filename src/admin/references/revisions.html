Settings

Revision displays are defined in the page settings. All revision pages must include the /assets/js/github.commits.widget.js and /assets/js/names.js scripts in order to be functional. Revisions display the lastest commits made to a specific branch in a specified repository. Pages can also display multiple repositories and branches at once.

Please note that the last property of any root, object, or array property must not be followed by a comma. This will cause our JSON validator to throw red flags and will cause the automatic builds to fail.

Name Description Value Default
"revisions": {} Holds different sets of data to control revision displays Object
Revisions Sub-properties
Name Description Value Default
"last": The amount of most recent commits to display Integer
"characters": The amount of characters to display in the primary commit message before it gets truncated Integer
"repositories": [] Defines each of the repositories to display Array
Example
"settings": {
    "revisions": {
        "last": 20,
        "characters": 1000,
        "repositories": [ ... ]
    }
}

Repositories Objects

Name Description Value Default
"user": The repository owner's GitHub username String
"repo": The repository name to display String
"branch": The specific branch of the repository to display String
Example
{
    "repositories": [
        {"user": "MCResourcePile", "repo": "overcast-maps-a-to-f", "branch": "master"},
        {"user": "MCResourcePile", "repo": "overcast-maps-g-to-n", "branch": "master"},
        {"user": "MCResourcePile", "repo": "overcast-maps-o-to-z", "branch": "master"}
    ]
}
