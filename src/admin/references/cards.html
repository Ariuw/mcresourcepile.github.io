Cards are used for portal index pages that link to other pages in a category or collection. These pages have no other content or purpose to them. Pages using cards must include the /assets/css/index.css stylesheet.

Please note that the last property of any root, object, or array property must not be followed by a comma. This will cause our JSON validator to throw red flags and will cause the automatic builds to fail.

Name Description Value Default
"cards": [] Defines each of the cards to display Array

Cards Objects

Name Description Value Default
"header": The title to display on the card String
"description": A description of the card and its location String
"url": The url or page name which the card will link to URL
"icon": The FontAwsome icon to display -- the icon fa-cubes would be entered as cubes for example String asterisk
"color": The color of the card -- can be red, blue, or left blank for the default String None
"light": Use a lightened variant of the selected color (does not work if no color specified) String false
Example
"cards": [
    {
        "header": "Overcast Collection",
        "description": "Publicly downloadable maps from the former Overcast Network map repository",
        "url": "overcast",
        "icon": "cube",
        "color": "red"
    }
]
