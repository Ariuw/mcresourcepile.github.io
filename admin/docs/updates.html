Update posts allow simple one-way communication between the ResourcePile staff and the users by creating simple blog-like posts regarding and changes, additions or announcements which may be of interest to readers. Update posts are not designed to be short descriptions of minor changes such as a map update as that it what the vision pages are for.

Update posts: src/data/updates.json
Main post attributes
Attribute Description Value Default
"heading": The title of the post. String
"id": The post's unique ID, which is used to reference it from other parts of the site. String
"important": Outlines the post making it more visible while also forcing it to the front page. Boolean false
"hidden": Hides the post from the site for archiving purposes Boolean false
"date": The date in which the post was written, and must be formatted as MM-DD-YYYY (eg. 01-29-2017). Date
"author": The post's author. Leaving this blank will make the post author ResourcePile. String
"content": The main posts content, accepts HTML elements. String
"buttons": [] Attribute containing button information - posts do not require buttons. Button Data false
Post buttons
Buttons are displayed at the end of the post and can link to important or relevant pages that were mentioned in the post content. Update posts do not require buttons.
Attribute Description Value Default
"details": The displaying text for the button String Learn More
"url": The URL which the button leads to. URL
